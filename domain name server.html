Domain Main Server

DNS is a hierarchical, distributed database system that translates human-readable domain names (e.g., example.com) into machine-readable IP addresses, essential for internet navigation.


How It Works

Queries flow recursively (stub resolver forwards to recursive resolver, then root → TLD → authoritative) or iteratively (step-by-step). EDNS expands UDP payloads beyond 512 bytes for larger responses. Caching uses TTL to reduce latency.


Record Types

  • A/AAAA: IPv4/IPv6 addresses.
  • CNAME: Aliases (e.g., www → example.com).
  • MX: Mail servers.
  • TXT: SPF/DKIM/DMARC for email security
  • SRV: Service locations.
  • CAA: Cert authority restrictions.

Threats and Mitigations

Cache poisoning (Kaminsky attack) forged by spoofing; mitigated by DNSSEC (cryptographic signatures chaining from root). DoS via amplification (BGP anycast distributes load). DoH/DoT encrypt queries.


Facts

Over 1,500 TLDs (ICANN-managed); 13 root server clusters (1,000+ instances)
